BrowserCoordinateKit Coordinate Transformations

This page contains visualizations of the mathematical coordinate transformations implemented in the BrowserCoordinateKit library.

Mathematical Overview Symbolic Validations

Screen to Normalized Transformation

Converts absolute screen coordinates to normalized [0,1] range

The screen-to-normalized transformation maps physical screen coordinates to a normalized [0,1] range, making them independent of screen resolution:

\(T_{S \to N}(p_s) = \left(\frac{x_s}{s_w}, \frac{y_s}{s_h}\right)\)

Where \(s_w\) and \(s_h\) are the screen width and height respectively.

Screen to Browser Transformation

Converts screen coordinates to browser-relative coordinates

The screen-to-browser transformation translates points from absolute screen coordinates to coordinates relative to the browser window:

\(T_{S \to B}(p_s) = p_s - b = (x_s - b_x, y_s - b_y)\)

Where \(b = (b_x, b_y)\) is the position of the browser window's top-left corner in screen coordinates.

Browser to Logical Transformation

Converts browser coordinates to logical coordinates (DPI scaling)

The browser-to-logical transformation converts physical pixels to logical pixels by applying the DPI scaling factor:

\(T_{B \to L}(p_b) = \frac{p_b}{\sigma} = \left(\frac{x_b}{\sigma}, \frac{y_b}{\sigma}\right)\)

Where \(\sigma\) is the DPI scaling factor (e.g., 2.0 for a high-DPI display).

Composite Transformation

Complete screen-to-logical transformation pipeline

The composite transformation combines screen-to-browser and browser-to-logical transformations:

\(T_{S \to L} = T_{B \to L} \circ T_{S \to B} = \frac{p_s - b}{\sigma} = \left(\frac{x_s - b_x}{\sigma}, \frac{y_s - b_y}{\sigma}\right)\)

This allows for direct conversion from screen coordinates to logical coordinates used by the browser.

These visualizations were generated using Python's matplotlib library to demonstrate the mathematical transformations implemented in BrowserCoordinateKit.
